核心概念

倒排索引（Inverted Index）

将长文本分词，把词项作为索引，对应原始文本 id 作为内容组织成的索引。会对词项按字典序排序，便于后续二分查找。

Term Index

不可能将整个倒排索引所有词项都加载到内存，Term Index 是一个字典树（Trie），每个节点保存它对应在倒排索引中的位置，用于快速定位词项。

词项（Term）

分词后得到的拆分内容。

Doc Values

想针对原始文本中的某个属性（例如时间）排序时，就把这个属性和 id 抽出来，单独排序，用空间换时间。Doc Values 是 ES 默认开启的列式存储结构。

Segment

是 Lucene 中完整的查询最小单元，每个 Segment 内部包含独立的倒排索引。多个 Segment 共同构成一个分片。

Stored Fields

原始文本字段，指 _source 中存储的原始 JSON 文档。

Segment Merge

可以对 Segment 并发读写，但 Segment 数量过多会耗尽文件句柄，因此需要定期合并，合并时会删除已标记为删除的文档。

Shard

分片，可基于文档 id 哈希路由，实现数据水平扩展。

Node

物理组织，一个 ES 节点是一个运行中的 ES 实例。

三大特性

  • 高性能：基于倒排索引、内存缓存、列式存储等机制保证快速检索
  • 高扩展性：通过分片机制实现水平扩展
  • 高可用：通过副本分片实现故障转移

集群选主

去中心化——不引入中心节点实现选主协调，基于 Raft 模块实现 Master 节点选举。

数据类型

  • 结构化数据：有固定 schema 的数据，如关系型数据库中的表
  • 非结构化数据：无固定格式的数据，如文本、日志
  • 半结构化数据：介于两者之间，如 JSON、XML

端口说明

  • 9300：ES 集群节点间通信端口（TCP）
  • 9200：HTTP REST API 访问端口

基础操作

创建索引

需要先使用 PUT 方法创建索引，不能用 POST，因为 PUT 是幂等的，多次执行结果一致；POST 语义是非幂等的，通常用于创建文档。

创建文档

  • 使用 POST 时，ES 自动生成文档 id
  • 使用 PUT 时，需指定文档 id，且是幂等的

复杂查询条件构建

SearchSourceBuilder

对整个查询请求体的抽象，可以添加查询条件、分页、排序、高亮、聚合等配置。

QueryBuilders

单个查询条件的构造工厂，可以构造多种查询类型。

BoolQueryBuilder

组合多个查询条件，支持 must、should、must_not、filter。

HighLightBuilder

高亮展示查询结果。

补充：POST 与 PUT 的语义区别

方法 语义 ES 中的使用场景
PUT 幂等，指定完整路径 创建索引、指定 id 创建/更新文档
POST 非幂等，可自动生成资源标识 自动生成 id 创建文档、部分更新
