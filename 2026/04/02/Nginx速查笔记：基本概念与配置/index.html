一、Nginx 是什么

Nginx 是一个高性能的 HTTP 服务器 / 反向代理服务器 / 负载均衡器。

它最核心的定位是：网关。也就是所有请求的“统一入口”。

二、Nginx 能做什么

1. Web 服务器

  • 托管前端静态资源（HTML、CSS、JS、图片）
  • 通过 root 或 alias 指定文件路径
  • 结合 try_files 实现 SPA 路由 fallback

2. 反向代理

  • 把请求转发给后端服务（Java、Node、Go、Python 等）
  • 对前端“隐藏”真实后端地址和端口
  • 可以挂多个后端（负载均衡）

3. 负载均衡

  • 通过 upstream 定义后端服务器集群
  • 支持多种负载策略：轮询、最少连接、IP 哈希等
  • 支持动态扩缩容

4. 网关能力

  • 限流、封禁 IP、修改请求/响应内容
  • 支持 HTTP、HTTPS、HTTP/2、WebSocket、TCP、UDP 等协议
  • 可插拔的模块化能力（压缩、缓存、日志等）

三、核心概念

概念 说明
master 进程 管理进程，负责读取配置、启动/停止/重启 worker，不处理实际请求
worker 进程 真正处理请求的进程，数量通常 = CPU 核数
多进程优势 进程间隔离，一个 worker 挂掉不影响其他，稳定性极高
事件驱动模型 每个 worker 可以同时处理成千上万个连接，无需多线程锁竞争
多进程共享端口 通过 SO_REUSEPORT 或 fork 继承，多个 worker 可以同时监听同一个端口

五、反向代理如何消除跨域

  • 浏览器只和 Nginx 交互（同源）
  • Nginx 在服务器内部把请求转给真正的后端
  • 浏览器看不到后端地址和端口 → 彻底绕过跨域

这是“前端同源 + 后端隐藏”的组合，本质是通过反向代理“欺骗”浏览器。

六、负载均衡 upstream

  • weight：权重
  • backup：备用服务器
  • 支持动态扩缩容，对外只暴露一个 Nginx 地址

七、HTTPS 配置

  • 80 端口和 443 端口可以配置在同一个 http 块中
  • 证书需要包含完整链（服务器证书 + 中间证书），浏览器内置根证书

八、location 匹配规则

写法 说明
location / 前缀匹配，优先级最低
location = / 完全匹配，优先级最高
location ~ 正则匹配（区分大小写）
location ~* 正则匹配（不区分大小写）
try_files 按顺序尝试文件/目录/fallback

九、rewrite 与重定向

  • permanent：301 永久重定向
  • redirect：302 临时重定向
  • last / break：内部重写

十一、学习建议

  1. 先理解“反向代理”和“负载均衡”两个核心场景
  2. 本地跑一个 Nginx + 两个简单后端服务，手动配置 upstream 和 proxy_pass
  3. 理解跨域问题本质 + 反向代理如何解决
  4. 理解多进程 + 端口共享的底层原理（不用太深，但要知道不是“master 分发请求”）
