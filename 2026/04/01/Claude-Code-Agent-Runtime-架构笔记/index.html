Claude Code Agent Runtime 架构笔记

一、Agent Loop（主循环）

核心流程

  1. 系统提示词组装：静态前缀 + DYNAMIC_BOUNDARY + 动态上下文，支持 Prompt Cache 边界控制
  2. 上下文治理：5层压缩机制，预算控制
  3. Claude API 流式：Streaming + 扩展思考
  4. Tool 调度执行：并发分组、权限校验、Hook 拦截
  5. 循环控制：续轮、终止、恢复，直到纯文本返回

关键设计原则

  • 消息驱动：用户输入、模型输出、tool 进度、系统事件全部是 Message，transcript 是恢复核心
  • 高效传递工具：避免污染提示词，尽量少消耗上下文
  • Prompt Cache 优先：工具稳定排序、DYNAMIC_BOUNDARY 分界、deferred tools、cache break detection

二、上下文治理（5层压缩）

层级 名称 策略
① snip compact 旧 tool_result → [snipped]，保留结构
② microcompact strip 旧 tool results，通知 cache detector
③ context collapse 折叠已完成子对话
④ autoCompact 阈值触发摘要
⑤ reactive PTL 错误触发压缩

记忆机制

  • MEMORY.md：项目级记忆，增大上下文但需压缩
  • compact boundary：压缩点 + 增量恢复
  • tool result 预算控制：避免上下文爆炸

子智能体隔离

  • 子任务交给子智能体，物理隔离上下文
  • 独立 tool pool / model / 权限 / worktree
  • 支持 SendMessage 团队协作

三、工具系统（40+ 工具）

注册中心（tools.ts）

  • 只注册工具名称，稳定排序
  • feature gate 控制
  • prompt cache 友好

并发执行（toolOrchestration.ts）

  • 并发安全分组
  • 读写分离并行执行

执行层（toolExecution.ts）

  • schema 校验（Zod v4）
  • hook 拦截（PreToolUse / PostToolUse）
  • 权限系统（4路竞争）
  • 遥测（OTel）

统一协议（Tool.ts）

  • call + checkPermissions + render + isReadOnly

流式执行（StreamingToolExecutor）

  • 流式中并发执行
  • 进度渲染

来源 说明

九、技术栈

十、关键架构判断（原文总结）

  1. 消息驱动：一切皆是 Message，transcript 是恢复核心
  2. Prompt Cache 优先：工具稳定排序、DYNAMIC_BOUNDARY 分界、deferred tools、cache break detection
  3. 四层扩展叠加：Command + Skill + Plugin + MCP 并行不冲突，决定这是平台而非客户端
  4. Agent 是一级概念：独立 tool pool / model / 权限 / worktree，支持 SendMessage 团队协作

这套架构的核心思想是：通过消息驱动、多层压缩、工具并发、扩展叠加，在有限的上下文预算内实现高效的智能体循环。 子智能体隔离和物理 worktree 是控制上下文爆炸的关键手段。
