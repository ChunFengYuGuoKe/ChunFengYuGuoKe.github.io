题目出自软件设计师2022年下半年第5题，题目：

  1. 做这道题的时候第一个困惑的点是数组的边界问题。因为一般来说，对于一个规模为n的数组，它的最大索引应该是n - 1，而在题目的代码中，却出现了j <= n和i = n这样的条件，这是不合常理的，那么假设数组的最大索引确实是n，则最小索引应该是1，思考一下就会发现，这样确实是合理且必须的，因为要用2 * i和2 * i + 1来表示i的左右子节点，就必须从1开始，从0开始是有问题的，如果从0开始，那么 2 * 0 = 0，它的子节点是它自己，这显然是错误的，2 * 0 + 1 = 1，显然也是错误的，一个元素的右子节点不可能是它紧挨的下一个元素！

  2. 好，既然第一个问题澄清了，元素是从索引1开始的，那么代码中多次出现的R[0]是什么作用呢？经过分析，R[0]起到了一个temp变量的作用，用于在交换数据的时候暂存其中一个即将被覆盖的值，也算是没有浪费数组空间了

  4. 为什么 n / 2 ~ n之间的元素不用建堆？因为堆是一个完全二叉树，n / 2 ~ n之间的元素是不会有子节点的，所以它们本身就是合法的堆，不用再以它们为根节点建堆了。

  5. 为什么 HeapSort 函数中建堆是从 n / 2开始往前建？从后往前建是因为在以更高层位置为根节点建堆时，必须保证以子节点为根节点的字数已经是合法堆，否则就会有错误。

  6. 为什么在每一轮排序之后还需要Heapify(R, 1, i - 1);并且排序的规模是i - 1？索引i的元素已经是归位了，问题的规模缩减了，并且原来索引i位置的元素交换到了堆顶，那这个时候就要对它重新入堆，给它找到一个合适的位置，维护整个堆的合法性。
