https://leetcode.cn/problems/find-k-closest-elements/description/
给定一个 排序好 的数组 arr ，两个整数 k 和 x ，从数组中找到最靠近 x（两数之差最小）的 k 个数。返回的结果必须要是按升序排好的。

整数 a 比整数 b 更接近 x 需要满足：

|a - x| < |b - x| 或者
|a - x| == |b - x| 且 a < b

解法一——先找到数组中距离x最近的元素的索引，再利用双指针向两边扩散

为什么是从int i = left + 1开始遍历添加，这里的left是什么含义？

left 和 right 的含义

在这个解法中：

  • left：窗口的左边界索引的前一个位置（开区间）
  • right：窗口的右边界索引（开区间）
  • 实际窗口：(left, right) 即从 left+1 到 right-1

text

为什么 for (int i = left + 1; i < right; i++)？

1. 开区间的设计

java

初始时，窗口 (p-1, p) 不包含任何元素：

  • left = p-1：窗口左边的元素（不包含）
  • right = p：窗口右边的元素（不包含）

2. 扩展过程

java

每次扩展时：

  • right++：向右扩展（包含更多右边的元素）
  • left--：向左扩展（包含更多左边的元素）

但注意：left 和 right 始终指向窗口之外

3. 最终的窗口表示

循环结束后：

  • 窗口包含 k 个元素
  • 窗口范围：(left, right)（开区间）
  • 实际元素：arr[left+1] 到 arr[right-1]

总结

在这个解法中：

  • left 和 right 是开区间的边界
  • 实际元素在 (left, right) 之间
  • 所以遍历时从 left+1 开始，到 right-1 结束

这种开区间的设计使得：

  1. 边界检查更简单
  2. 窗口大小计算更方便
  3. 扩展操作更统一

这是一个很巧妙的设计，体现了区间表示法的灵活性。虽然最终结果相同，但这种思考方式在某些问题中会更简洁。

left 的最终含义

最终计算出的 l 是：长度为 k 的最佳连续子数组的起始索引

也就是说：

  • arr[l] 到 arr[l + k - 1] 就是我们要找的 k 个最接近 x 的数
  • 这个窗口是数组中所有长度为 k 的连续子数组中，最接近 x 的那一个

为什么 l 具有这个含义？

1. 二分查找的搜索目标

我们在搜索的是：长度为 k 的滑动窗口的起始位置

二分查找的范围：l = 0, r = n - k - 1 = 4

2. 决策条件的目标

我们比较的是：

这个比较在判断什么？

  • x - arr[mid]：窗口左边界到 x 的距离
  • arr[mid + k] - x：窗口右边界之外第一个元素到 x 的距离

关键洞察：
如果 x - arr[mid] > arr[mid + k] - x，说明：

  • 窗口的左边界离 x 太远
  • 窗口的右边界之外有更接近 x 的元素
  • 所以整个窗口应该向右移动

3. 二分查找结束时 l 的位置

二分查找的终止条件是 l > r。结束时：

  • l 指向第一个不满足向右移动条件的起始位置
  • 也就是说，l 是使得窗口不能再向右移动的最小起始位置

数学证明为什么 l 是最佳起始位置

定义函数 f(i)

设 f(i) 表示以 i 为起始的窗口的”偏向性”：

性质：

  1. 单调性：因为数组有序，arr[i] 和 arr[i+k] 都随 i 增大而增大
    • 所以 f(i) 随 i 增大而减小
  2. 零点性质：当 f(i) > 0 时，窗口应向右移；f(i) <= 0 时，窗口合适

最终 l 的特性证明

定理：

最终得到的 l 满足：

  1. 对于所有 i < l，以 i 为起始的窗口不如以 l 为起始的窗口好
  2. 对于所有 i > l，以 i 为起始的窗口也不如以 l 为起始的窗口好

证明：

  1. 对于 i < l：

    • 在二分过程中，当检查 i 时，我们发现它应该向右移动
    • 说明 x - arr[i] > arr[i+k] - x
    • 这意味着窗口太靠左，右边界之外有更接近 x 的元素

  2. 对于 i > l：

    • 二分结束时 l 是第一个不满足向右移动条件的位置
    • 所以对于 i = l+1, l+2, ...，虽然我们没有显式检查，但由单调性可知它们更不合适

直观理解：滑动的平衡点

想象 x 是一个磁铁，窗口是 k 个铁球：

  • 如果窗口太靠左，右边的铁球会更想靠近磁铁
  • 如果窗口太靠右，左边的铁球会更想靠近磁铁
  • l 就是平衡点：窗口再往左或往右都不会让整体更接近磁铁

总结

l 的含义是经过精心设计的二分查找后得到的：

  1. 它是目标：我们要找的就是这个起始位置
  2. 它是最优的：数学上可以证明没有其他起始位置更好
  3. 它是明确的：二分查找精确地定位到了这个位置

这种算法的美在于：将寻找 k 个离散点的问题转化为寻找一个连续窗口的问题，然后利用二分查找高效解决。l 就是这个转化思想的结晶。
